상용구 추가 워크플로우를 실행합니다.

추가 대상 키: $ARGUMENTS

시작 전 반드시 아래 순서로 파일을 읽습니다:

1. `lib/ai/resources/template-list.json` — 중복 및 기존 항목 확인
2. `lib/ai/resources/template-categories.json` — 대분류 목록 확인
3. `lib/ai/resources/cc-list.json` — 사용자가 언급한 C.C. 존재 여부 확인용
4. `ai-docs/pending-matches.md` — 대기 커넥션 확인 (섹션 2: C.C.→상용구 대기)
5. `ai-docs/templates/chest-pain/template.json` — 템플릿 구조 참조 (fields, pe, history 섹션 구조 포함)
6. `ai-docs/templates/chest-pain/schema.json` — 스키마 구조 참조

이후 CLAUDE.md "상용구 추가 절차"를 단계별로 따릅니다.

**대분류(카테고리) 확인:**

파일 읽기 후, 반드시 사용자에게 명시적으로 질문합니다:
"이 상용구는 어느 분류에 넣을까요? `template-categories.json`에 등록된 분류 목록을 보여주고, 번호로 선택하게 합니다. 선택된 값을 `template-list.json`의 해당 항목 `category` 필드에 저장합니다."

**분류 내 순서 확인:**

카테고리 선택을 받은 뒤, 해당 분류에 이미 존재하는 상용구 목록을 번호와 함께 보여주고, 반드시 사용자에게 추가로 질문합니다:
"이 분류에 현재 등록된 상용구 순서입니다. 새 상용구를 어느 위치에 배치할까요? 마지막에 추가하려면 '마지막'이라고, 특정 순서에 넣으려면 몇 번째 뒤에 넣을지 알려주세요."
사용자가 선택한 위치를 기준으로 `template-list.json`에서 해당 분류 항목들 사이의 올바른 위치에 새 항목을 삽입합니다.

**C.C. 커넥션 확인:**

파일 읽기 후, 반드시 사용자에게 명시적으로 질문합니다:
"이 상용구를 연결할 C.C.가 있나요? (있으면 C.C. 이름을 알려주세요. 없으면 '없음'이라고 답해주세요.)"

**커넥션 처리 흐름:**

- 사용자가 C.C.를 답한 경우:
  - cc-list.json에 해당 C.C. 존재 →
    1. 해당 C.C.의 현재 templateKeys 목록과 순위를 표시합니다. (예: `[chest-pain (0순위), chest-pain-v2 (1순위)]`. 없으면 "현재 연결된 상용구가 없습니다.")
    2. "이 상용구를 몇 순위로 연결할까요? (숫자 입력, 0순위는 C.C. 입력 시 자동 로드됩니다)"
    3. 0순위 충돌 시: "0순위는 이미 {key}로 설정되어 있습니다. 0순위로 설정하면 기존 0순위가 1순위로 밀립니다. 계속할까요?"
    4. 해당 C.C.의 templateKeys에 `{ "key": "...", "rank": N }` 형태로 추가. 0순위 충돌 수락 시 기존 0순위 항목을 1순위로 재조정.
  - 존재하지 않음 → 순위를 먼저 물어본 후, pending-matches.md 섹션 4(상용구→C.C. 대기)에 rank 정보와 함께 기록 (중복 확인 후)
- 사용자가 '없음'이라고 답한 경우: 아무것도 기록하지 않음

**커넥션 순위 공통 규칙:**

- 0순위는 C.C.당 상용구 최대 1개
- 0순위로 지정된 상용구는 C.C. 입력 시 자동 로드됨
- pending-matches.md에 기록할 때도 rank 정보를 함께 기록 (예: `rank: 0`)

**대기 커넥션 확인:**

- pending-matches.md 섹션 2에서 이 templateKey를 기다리는 C.C.가 있으면 → 연결 여부 질문
- 수락: cc-list.json 업데이트 + 해당 행 삭제 / 거절: 해당 행 삭제만

**고정 주석 줄(`#`) 처리:**

사용자가 제공한 양식에 `#`으로 시작하는 줄이 있으면, 해당 줄을 `output_example`에 `#` 포함 상태로 그대로 저장합니다.
`#`은 "AI가 이 줄을 수정하지 말 것"을 의미하는 마커입니다.

- AI 차팅 생성 시: `#`을 제거하고 내용을 그대로 출력
- 프론트엔드 템플릿 미리보기 시: `#`을 제거하고 표시
- `output_example` 파일 내에는 `#` 포함 상태로 보존 (AI가 어떤 줄이 고정인지 인식하기 위함)

**AI 생성 대체 줄(`$`) 처리:**

사용자가 제공한 양식에 `$`으로 시작하는 줄이 있으면, 해당 줄을 `output_example`에 `$` 포함 상태로 그대로 저장합니다.
`$text`는 "AI가 이 설명에 맞게 자유 형식으로 채울 것"을 의미하는 마커입니다.

- AI 차팅 생성 시: `$text`를 삭제하고 설명에 맞는 자유 형식 서술로 대체. 정보가 없으면 `$` 기호만 제거하고 원문 텍스트를 그대로 출력
- 프론트엔드 템플릿 미리보기 시: `$`을 제거하고 표시
- `output_example` 파일 내에는 `$` 포함 상태로 보존 (AI가 어떤 줄을 생성할지 인식하기 위함)

**반드시 사용자에게 순서대로 요청해야 하는 양식:**

1. P.I. template 양식 (현병력 구조화 필드 + output_example) → fields 섹션에 저장
2. History 양식 (기본값 상태의 출력 포맷) → history 섹션에 저장
3. P/E 양식 (기본값 상태의 출력 포맷) → pe 섹션에 저장
4. 케이스 예시 데이터 (ai-docs/templates/{templateKey}/examples.md에 저장)

>
>

예시를 받으면 `ai-docs/templates/{templateKey}/examples.md`에 append합니다. 파일이 없으면 새로 생성합니다.
