기존 상용구에 케이스 예시를 추가하는 워크플로우를 실행합니다.

대상 키: $ARGUMENTS

**사전 확인:**

시작 전 아래 두 가지를 확인합니다:

1. `lib/ai/resources/template-list.json`에 해당 templateKey가 존재하는지 확인
2. `ai-docs/templates/{templateKey}/` 폴더가 존재하는지 확인

둘 중 하나라도 없으면: "'{templateKey}' 상용구를 찾을 수 없습니다. templateKey를 확인하거나 /tmpl 명령어로 먼저 상용구를 등록해 주세요."라고 안내하고 중단합니다.

---

**케이스 예시 수집:**

사용자에게 아래 안내 문구와 함께 예시를 요청합니다:

> 추가할 케이스 예시를 아래 4섹션 형식으로 제공해 주세요. 여러 케이스를 한 번에 제공해도 됩니다.
>
> ```
> # Case N
> ## HPI (줄글)
> (AI가 생성한 또는 직접 작성한 HPI 줄글 예시)
>
> ## P.I template
> (실제 출력 예시)
>
> ## History
> (실제 출력 예시)
>
> ## P/E
> (실제 출력 예시)
> ```

---

**examples.md 저장:**

예시를 받으면 `ai-docs/templates/{templateKey}/examples.md`에 append합니다.

- 기존 파일이 있으면: 마지막 케이스 번호 이후로 이어서 작성 (번호 자동 증가)
- 파일이 없으면: 새로 생성

---

**pruning 플로우 (케이스 수 > 10개일 때):**

저장 후 총 케이스 수를 확인합니다. 11개 이상이면 아래 절차를 진행합니다:

1. 전체 케이스를 읽고 아래 기준으로 품질 평가:
   - **다양성**: 증상 양상·중증도·시간 경과가 다른 케이스 우선
   - **완성도**: 4섹션이 충실히 채워진 케이스 우선
   - **중복 제거**: 내용이 거의 동일한 케이스는 1개만 유지

2. 유지할 10개를 선정하여 사용자에게 아래 형식으로 제시:

   > 현재 케이스가 N개입니다. 품질 기준으로 아래 10개를 유지하도록 추천합니다:
   >
   > - Case 1: (한 줄 요약)
   > - Case 3: (한 줄 요약)
   > - ...
   >   제외: Case 2 (Case 1과 유사), Case 7 (4섹션 미완성) ...
   >
   > 이대로 진행할까요? (승인 / 거절 — 거절 시 남길 케이스 번호를 직접 지정해 주세요)

3. **승인 시**: 선정된 10개만 남기고 번호를 1부터 재정렬하여 examples.md 덮어쓰기
4. **거절 시**: 사용자가 지정한 케이스 번호로 동일하게 처리

---

**메타 학습 분석 step:**

저장이 끝나면 방금 추가된 예시를 다시 읽고 아래 4개 프롬프트 파일에서 보완 가치가 있는 패턴이 있는지 분석합니다:

- `ai-docs/prompts/generate-pi.md`
- `ai-docs/prompts/generate-template.md`
- `ai-docs/prompts/generate-history.md`
- `ai-docs/prompts/generate-pe.md`

분석 결과:

- **새 패턴 발견 시**: 변경안(diff 형식)을 사용자에게 제시 → 승인 시 해당 파일 업데이트, 거절 시 건너뜀
- **새 패턴 없음**: "기존 프롬프트와 잘 일치합니다. 추가 변경 없음."으로 보고
