# ER Scribe MVP PRD

## 🎯 핵심 정보

**목적**: 응급실 의사가 모바일로 문진 키워드를 빠르게 입력하면 AI가 병원 스타일의 차팅 초안 2종(HPI 줄글 + C.C 상용구)을 자동 생성해주는 어시스턴트
**사용자**: 응급실에서 모바일로 환자를 문진하고, 데스크톱으로 차트를 작성하는 응급의학과 의사

---

## 🚶 사용자 여정

```
1. 홈 페이지
   ↓ 로그인 버튼 클릭 (비로그인 시)

2. 로그인 페이지
   ↓ 인증 성공 → 자동 리다이렉트

3. 대시보드 페이지
   ↓ [+ 새 케이스] 버튼 클릭

4. 새 케이스 입력 페이지 (모바일)
   ↓ C.C 입력 (자동완성)
   ↓ 키워드 카드 입력 (반복)
   ↓ [차팅 생성] 버튼 클릭

5. 케이스 결과 페이지 (데스크톱)
   → [생성 중] → 폴링 → 결과 표시
   → [완료] → HPI 줄글 + 상용구 차팅 표시
   → 편집 후 복사 → EMR 붙여넣기

6. (선택) 케이스 목록 페이지
   ↓ 과거 케이스 클릭 → 케이스 결과 페이지로 이동
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명                   | 설명                                                                          | MVP 필수 이유                          | 관련 페이지                               |
| -------- | ------------------------ | ----------------------------------------------------------------------------- | -------------------------------------- | ----------------------------------------- |
| **F001** | 케이스 생성 및 카드 입력 | 새 케이스를 만들고 자유 텍스트 키워드를 카드 형태로 입력. 입력 즉시 저장      | 핵심 데이터 수집 수단                  | 새 케이스 입력 페이지                     |
| **F002** | C.C 자동완성             | 사전 정의된 C.C 목록에서 자동완성 추천. 상용구 존재 여부 배지 표시            | C.C 정확도가 AI 생성 품질에 직결       | 새 케이스 입력 페이지                     |
| **F003** | 시간 태그 자동 감지      | "3일 전", "2시간 전" 등 상대 시간 표현을 감지해 카드에 배지 부착 및 자동 정렬 | 시간 순서가 HPI 품질의 핵심            | 새 케이스 입력 페이지                     |
| **F004** | AI 차팅 생성             | 입력 카드를 3단계(정규화 → HPI → 상용구)로 처리해 차팅 초안 2종 생성          | 제품의 핵심 가치                       | 새 케이스 입력 페이지, 케이스 결과 페이지 |
| **F005** | 차팅 편집 및 복사        | 생성된 HPI와 상용구를 편집하고 독립 복사 버튼으로 클립보드 복사               | 의사가 실제로 EMR에 붙여넣는 최종 단계 | 케이스 결과 페이지                        |
| **F006** | 케이스 목록 조회 및 삭제 | 본인의 케이스를 날짜·C.C·상태별로 목록으로 보고 삭제                          | 이전 케이스 재접근 및 관리             | 케이스 목록 페이지, 대시보드 페이지       |
| **F007** | 수동 상용구 선택         | C.C가 목록에 없을 때 다른 상용구 템플릿을 수동으로 선택                       | 목록 외 C.C에서도 상용구 활용 가능하게 | 케이스 결과 페이지                        |

## 📌 주요 설계 결정 사항

1. **원본 텍스트 보존**: `case_inputs.raw_text`는 절대 수정하지 않음. AI 생성 JSON은 중간 표현이며, 원문이 진실의 원천
2. **재생성 히스토리**: `case_results`는 케이스당 다수(1:N). `cases.current_result_id`로 현재 표시본 지정. 이전 결과 유지로 재생성 후 비교 가능
3. **AI 파이프라인 분리**: 입력 정규화 → HPI 생성 → 상용구 생성을 3단계로 분리해 각 단계 독립 재생성 및 프롬프트 관리 가능
4. **structured_schema 확정 전까지 placeholder 진행**: 스키마는 실제 케이스 예시를 ChatGPT로 분석 후 확정 예정
